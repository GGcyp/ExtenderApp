# 贡献指南（CONTRIBUTING.md）

## 概述
欢迎为本项目贡献代码。为保证代码一致性与高可维护性，请在提交前遵循下列规则。

## 主要规则
- 所有公共类型（类、结构体、接口、枚举）和公共成员（方法、属性、字段（若为 public）、事件、操作符等）必须包含 XML 文档注释（中文）。注释应准确描述用途、参数、返回值和可能抛出的异常。
- 私有成员可根据需要添加注释，但非强制。
- 保持现有代码风格与格式；在提交前运行代码格式化。项目使用 .editorconfig 定义格式化规则。
- 使用有意义的命名，遵循 .NET 命名约定（PascalCase 用于类型和公共成员，camelCase 用于私有字段/参数等）。
- 为公共 API 写单元测试或集成测试（如适用）。

## 提交与审查
- 每个功能或修复请使用单独分支提交，并在 PR 描述中说明变更动机与影响范围。
- 提交前确保本地通过编译与单元测试；CI 也会执行相同检查。
- PR 应包括自检清单，至少包含：编译通过、测试通过、格式化已应用、没有未处理的 TODO。

## 文档注释要求
所有 public 类型、public 成员（方法、属性、事件、构造函数）、以及 public 的泛型参数行为要有 XML 文档注释。注释应使用 `<summary>` 描述用途，必要时使用 `<param name="...">`、`<returns>`、`<exception cref="...">`。对于涉及内存/池/引用清理的类型，请说明清理策略（何时清零，何时不清零，使用 RuntimeHelpers.IsReferenceOrContainsReferences<T>() 判定等）。

## 工具与检查
- 推荐在 Visual Studio 中启用代码格式化和分析规则（例如：__Tools > Options__ 中的格式化设置），并在 CI 中运行 `dotnet format` 与 `dotnet build`。
- 使用 `StyleCop` / `Roslyn Analyzers` 可选项以保证代码质量。

## 其他
- 对于公共 API 的变更请在 PR 中说明是否为破坏性变更，并提供兼容性建议。
- 任何不确定的规范可以在 PR 中与维护者讨论。
